# Video Reference Frames Playbook

## 目标

生成服务图生视频的首帧、尾帧和关键帧 Prompt，让静态图真正能承担视频锚点作用。

## 先判帧类型

- 首帧图：负责开场状态和起始构图
- 尾帧图：负责收尾状态和目标落点
- 关键帧图：负责中途爆点、转折或动作中段

## 资产清单维度

### 主体锚点

- 脸部识别点
- 发型
- 服装
- 情绪底色

### 场景锚点

- 主场景结构
- 主道具
- 光源方向
- 色调

### 运动预留

- 哪些部位可动
- 哪些道具可动
- 运动方向
- 镜头起步方向

### 镜间关系

- 当前帧承接上一帧什么信息
- 当前帧要引出下一帧什么动作
- 哪些元素必须跨帧稳定

## 什么时候继续下钻控制页

如果参考帧目标明确追求下面这些结果，不要把它当普通海报图来写：

- `像真实照片`
- `像手机或相机抓拍`
- `办公室 / 会议室 / 工位纪实`
- `漂亮侧颜`
- `自然体态`
- `不要塑料皮和网红感`

优先下钻：

- `style-control/anti-ai-realism.md`
- `style-control/failure-patterns-negative.md`
- `portrait/side-profile-bodyline.md`
- `scene-office/office-candid-photography.md`

使用原则：

- 首帧要先保证真实锚点成立，再谈后续运动
- 如果参考帧本来就要服务图生视频，人物皮肤、体态和空间证据越真实，后续跨帧越稳
- 办公室纪实类参考帧要保留普通灯光、环境道具和非海报式构图，给后续运动留真实空间

## 推荐输出包

1. 首帧图
2. 尾帧图
3. 1-3 张关键过渡帧
4. 每帧对应的保留项说明

## 输出骨架

```text
帧类型：
必须稳定的锚点：
允许变化的元素：
可运动元素：
当前帧的叙事职责：
参考帧 Prompt：
跨帧限制：
```

## 使用原则

- 首帧重起点，尾帧重结果，关键帧重转折
- 参考帧必须显式写出一致性约束
- 关键帧只负责节拍转折，不负责交代全部剧情

## 常见失败

- 把参考帧画成普通美图，没有运动预留
- 帧与帧之间人物、服装、背景断裂
- 没有区分首帧、尾帧、关键帧的不同任务

## 修复方式

- 像海报不像帧：补“可运动元素”和“镜头起步方向”
- 帧间断裂：把稳定项显式写出来
- 关键帧没节拍感：补当前帧承上启下职责
- 参考帧太像精修美图：补真实感、失败模式和场景/体态控制页
