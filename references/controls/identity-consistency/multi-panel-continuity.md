# 多格叙事连续性控制

## 解决什么问题

在漫画页、条漫、多格故事板和连续关键帧中维护角色、场景、道具、动作、对白和剧情状态，避免每格像独立生成。

## 什么时候读取

- 漫画、条漫、四格、多格分镜或连续动作；
- 同一角色跨格变脸、换衣、左右反转；
- 道具数量、持有者和损坏状态漂移；
- 对白说话者和气泡位置混乱；
- 上一格动作结果无法承接下一格。

## 连续性状态表

每格至少记录：

```text
角色身份与外观
服装和配饰
人物左右、前后和朝向
动作输入状态
本格动作或事件
动作输出状态
道具持有者、位置和状态
场景区域与光线
表情、伤痕和剧情累计状态
对白说话者与阅读顺序
```

## 固定项与累计项

### 固定项

- 脸型、五官比例和发型主轮廓；
- 体型和身体比例；
- 主服装结构和基础配色；
- 场景门窗、道路、家具和主要道具；
- 漫画线稿、上色和阴影方式。

### 累计项

- 伤痕、污渍、湿发和服装破损；
- 道具打开、损坏、消耗或转移；
- 人物位置和朝向；
- 表情与情绪强度；
- 时间、天气和环境变化。

累计项必须从前一格继承，不能自动恢复。

## 输入与输出状态

每格采用：

```text
上一格输出
→ 本格变化
→ 本格输出
```

例如：

```text
输入：人物 A 右手持杯，站在桌左侧。
变化：A 把杯递给人物 B。
输出：B 左手接住杯，A 右手空出，两人位置不变。
```

不要只写“递杯子”，否则持有关系和手部状态容易漂移。

## 空间和方向

- 区分场景左 / 右与当前画面左 / 右；
- 建立人物 A / B 的相对位置；
- 运动方向跨格保持；
- 需要换轴时使用环境格、正面格或明确过渡格；
- 条漫滚动很长时，定期重复空间锚点；
- 场景切换必须用新场景建立格。

## 对白连续性

- 每段对白绑定明确说话者；
- 气泡尾巴和人物位置一致；
- 阅读顺序与分格顺序一致；
- 同一人物的气泡风格保持一致；
- 旁白、内心独白和对白使用不同框型；
- 长对白优先后期排版，生成阶段保留位置和说话关系。

## 多角色管理

- 使用稳定角色代号 A / B / C；
- 每格只突出必要角色；
- 群像先固定核心组、反应组和背景组；
- 角色离场后不要在下一格无解释出现；
- 新角色登场先用明确建立格确认身份和位置。

## Prompt 结构

```text
保持多格连续性：
固定 [角色身份、发型、服装、体型、场景结构和渲染方式]。
第 1 格输入状态为 [状态]，输出为 [状态]；
第 2 格承接第 1 格输出，仅改变 [动作或事件]；
逐格记录 [位置、朝向、道具、伤痕和情绪] 的累计变化。
对白绑定明确说话者，阅读顺序与分格一致。
禁止无原因换装、变脸、左右反转、道具复原和场景重构。
```

## 常见失败与修复

- 每格变脸：重复一级身份锚点并降低表情夸张；
- 道具突然复原：把损坏、消耗和持有状态设为累计项；
- 左右反转：固定场景原点和运动方向；
- 动作跳跃：写清输入、变化和输出；
- 气泡指错人：绑定说话者、位置和尾巴方向；
- 场景像换地点：重复门窗、家具、道路和光向锚点；
- 伤痕忽有忽无：逐格维护剧情累计状态。
