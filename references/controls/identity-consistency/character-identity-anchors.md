# 角色身份锚点与稳定区

## 解决什么问题

把“保持同一个角色”拆成可观察、可固定和可验证的身份锚点，避免角色在三视图、多角度、表情、换装和跨场景中发生变脸、变体型或核心特征漂移。

## 什么时候读取

- 角色资产需要三视图、多角度头像、表情板或换装板；
- 多图或多镜头中必须保持同一人物；
- 用户反馈“每张图像不同的人”；
- 角色换表情、服装、姿态或场景后身份漂移；
- 需要明确哪些特征固定、哪些特征允许变化。

## 身份锚点层级

### 一级锚点：决定“是不是同一个人”

- 年龄感与性别表达；
- 脸型和下颌轮廓；
- 眼距、眼型和眉眼关系；
- 鼻梁、鼻尖和鼻翼比例；
- 唇形、唇厚和人中关系；
- 颧骨、下巴和耳位；
- 发际线、刘海或分线；
- 头身比、肩胯关系和体型轮廓。

一级锚点跨图默认固定。

### 二级锚点：强化角色辨识

- 发色与发型主轮廓；
- 常态眼神和气质；
- 主服装轮廓和核心配色；
- 标志性配饰、伤疤、痣、纹身或道具；
- 鞋履、手套、帽子等高识别元素。

二级锚点可按任务选择固定强度。

### 三级变量：允许变化

- 表情；
- 姿态和动作；
- 局部妆面强度；
- 临时服装与场景；
- 光影、色调和镜头；
- 临时小道具；
- 剧情状态，如战损、淋雨或疲惫。

三级变量不能覆盖一级锚点。

## 脸部稳定区

角色资产中建议明确：

```text
稳定区：脸型、眼距、鼻唇比例、下颌、发际线
可变区：眉峰、眼睑、嘴角、面部肌肉和妆面强度
```

表情越强，越需要重复稳定区；不要依赖“same character”单句完成一致性。

## 身体稳定区

至少固定：

- 头身比；
- 肩宽与胯宽；
- 胸腰胯主要轮廓；
- 手臂和腿部长度关系；
- 身高感和体型；
- 习惯站姿与重心倾向。

不同姿态不能改变骨架比例。

## 换装控制

换装时分离：

```text
人物身份
≠ 服装身份
```

保持人物的脸部、发型主锚点、体型和常态气质；只替换服装轮廓、材质、配色和配饰。

如果发型也需要变化，应保留发际线、发色或其他强身份锚点。

## 推荐出图顺序

```text
1. 中性背景角色识别主图
2. 正、侧、背三视图
3. 4–6 个多角度头像
4. 基础表情组
5. 常用姿态与全身中性站姿
6. 手部、鞋履、配饰和固定道具
7. 换装组
8. 特殊剧情状态
9. 场景化主视觉
```

先稳定身份，再增加变量。

## Prompt 结构

```text
同一角色，固定：
[年龄感与身份]，
[脸型、眼距、眼型、鼻唇比例、下颌和耳位]，
[发际线、分线、发色和发型主轮廓]，
[头身比、肩胯关系和体型]，
[标志性配饰或道具]。

本张允许变化：
[表情 / 姿态 / 服装 / 场景 / 光影]。

禁止变化：
[核心脸部比例、骨架比例、发色或其他身份锚点]。
```

## 常见失败与修复

- 只写角色名字或 same character：补可观察身份锚点；
- 表情板变脸：降低表情强度，重复脸部稳定区；
- 换装像换人：把人物身份和服装变量分开；
- 三视图比例不一致：固定头身比、肩胯和服装长度关系；
- 发型变化导致身份丢失：保留发际线、发色或标志性配饰；
- 场景化主视觉才像角色：先用中性背景验证身份能独立成立。
