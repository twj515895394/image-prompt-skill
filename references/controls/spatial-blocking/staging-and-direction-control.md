# 站位、动线与空间方向控制

## 解决什么问题

把人物、道具、出入口和镜头之间的空间关系固定下来，避免多图、多镜头或多格故事板中左右方向反复、人物位置漂移、动线不可执行和道具突然换位。

## 什么时候读取

- 场景资产需要俯视站位图、动线或镜头入口；
- 双人、多人互动需要明确前后左右和视线；
- 分镜跨镜头保持空间连续；
- 用户要求三角站位、对峙、群像或动作路径；
- 生成结果出现左右反转、人物穿越空间或道具位置漂移。

## 建立空间原点

先固定：

```text
空间原点
+ 面向方向
+ 画面左 / 右
+ 前景 / 中景 / 背景
+ 出入口
+ 主活动区
```

例如：

```text
以房间中央桌子为原点，门在画面左后方，窗户在画面右侧，人物 A 位于桌前左侧，人物 B 位于桌后右侧。
```

多镜头中不要只写“左边、右边”，应说明是场景方向还是当前画面方向。

## 站位关系

### 双人关系

常用结构：

- 并排：亲近、合作、共同观看；
- 面对面：对话、谈判、冲突；
- 前后错位：权力差、跟随、保护或观察；
- 过肩关系：建立双方视线和信息差；
- 一坐一站：强化身份、压力或照顾关系。

### 三角站位

三人或“人物 + 人物 + 关键道具”可使用三角关系：

```text
主角为第一视觉点
→ 对手或搭档为第二点
→ 关键道具 / 出入口 / 第三人为第三点
```

三点不要完全等距；通过距离、朝向和明暗建立主次。

### 群像

先分组，再排人：

- 核心人物组；
- 反应人物组；
- 背景群众组；
- 功能性道具或空间节点。

不要把所有人平均排成一排。

## 视线控制

每个主要人物应明确：

- 正在看谁或什么；
- 是否避开对方视线；
- 视线是否指向画外信息；
- 视线与身体朝向是否一致；
- 观众是否能读懂视线目标。

多人场景中，视线可以形成闭环、对冲或单向压迫。

## 动线控制

```text
起点
→ 路径
→ 经过的空间节点
→ 终点
→ 镜头是否能完整看到
```

动作路径需要避开墙体、桌椅和其他人物，不要只写“向前走”。

复杂动作优先拆成：

```text
准备位置
→ 运动方向
→ 接触或转折点
→ 结果位置
```

## 镜头入口

场景资产至少考虑 2–4 个稳定镜头入口：

- 总览入口：交代空间；
- 关系入口：拍人物互动；
- 动作入口：完整看到运动路径；
- 细节入口：拍关键道具或局部事件。

镜头入口不能改变门窗、主道具和活动区的固定位置。

## 跨镜连续性

逐镜检查：

- 人物 A / B 的左右和前后关系；
- 运动方向是否保持；
- 视线是否连接；
- 道具持有者和落点；
- 出入口与场景区域；
- 上一镜动作终点是否等于下一镜起点；
- 光源方向和时间状态是否突然变化。

如需改变轴线或方向，应使用明确的过渡镜头建立新方向。

## Prompt 结构

```text
以 [空间原点] 为基准，
[出入口、窗户、主道具] 位于 [固定方向]。
人物 A 位于 [位置]，面向 [目标]；
人物 B 位于 [位置]，与 A 保持 [距离和角度]；
关键道具位于 [位置]。
动作从 [起点] 沿 [路径] 到达 [终点]，
镜头从 [入口] 能完整看到 [关系或动作]。
```

## 常见失败与修复

- 左右方向反复：固定空间原点并区分场景方向与画面方向；
- 人物像贴纸一样排开：建立距离、角度、前后和视线关系；
- 动作穿过家具：补路径和空间节点；
- 道具突然换位：逐镜记录持有者与落点；
- 俯视图与透视图不一致：先锁定门窗、主道具和活动区；
- 群像没有重点：按功能分组，建立第一、第二和第三视觉点。
