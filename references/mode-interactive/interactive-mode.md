# Interactive Mode

## 目标

在用户需要共创时，通过 `grill-me` 式持续追问把方向真正锁定，再输出更贴合意图的 Prompt。

## 硬规则

1. 每次只问一个问题。
2. 不设固定轮数上限。
3. 如果某个问题可以通过现有文字、图片或上下文推断，直接推断，不追问用户。
4. 只追最影响结果的那个变量，不开并行问卷。
5. 允许指出用户表达中的冲突，而不是把冲突默默平均掉。

## 起手方式

### 用户方向还很散

先给 `2-3` 个方向假设，再让用户挑或改：

- 更偏情绪写真
- 更偏角色海报
- 更偏叙事瞬间

### 用户方向已经很明确

直接追问当前最大的缺口，例如：

- “这张图你更想锁脸，还是更想换气质？”
- “这个角色你最不能变的是脸、发型还是服装？”
- “这组分镜里，你最想让观众先看动作、关系还是环境？”

## 问题优先级

### 成片生图

- 创作意图
- 主体气质
- 观看距离
- 光影情绪
- 风格强化

### 角色资产

- 固定身份
- 固定外貌
- 固定服装配色
- 表情包结构
- 可变项边界

### 场景资产

- 场景承担什么叙事任务
- 人物怎么进入空间
- 主道具是什么
- 镜头入口在哪里
- 光影氛围怎么服务叙事

### 分镜资产

- 这一镜的核心任务
- 上一镜承接什么
- 下一镜要引出什么
- 动作阶段怎么拆
- 哪个元素必须跨镜稳定

### 视频参考帧

- 这是首帧、尾帧还是关键帧
- 哪些元素必须稳定
- 哪些部位必须可动
- 这一帧承上启下的职责是什么

## 收口条件

满足以下任一条件即可停止追问：

- 用户明确要求出结果
- 再问只会做微调，主方向不再改变
- 主体、场景、镜头、光影、限制五层已经可以稳产出

## 最终输出

```text
方向决策摘要：
主方案 Prompt：
备选方案 A：
备选方案 B：
锁定变量：
剩余由系统自动补全的变量：
```

## 反模式

- 不要机械限制自己只问三轮
- 不要一开始就把完整 Prompt 砸给用户
- 不要一边说共创，一边默默替用户做关键方向决策
