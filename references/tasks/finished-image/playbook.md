# 成片生图 Playbook

## 用途

生成一张高完成度结果图，而不是资产清单、风格词列表或灵感摘要。

## 何时读取

适用于：

- 人物写真或叙事瞬间
- 产品图、电商图和产品主视觉
- 海报、封面和宣传画面
- 电影感画面与概念图
- 动漫、漫画、插画和 3D 成片

## 最少需要锁定

- 主体是谁或是什么
- 画面要表达什么事件、状态或情绪
- 场景在哪里
- 观众以什么距离和角度观看

缺少的信息如果能够稳妥推断，可以自动补全；如果缺失会改变题材或主方向，再进入交互模式。

## Prompt 组装顺序

```text
主体身份与辨识点
→ 场景与空间关系
→ 动作、状态或事件瞬间
→ 镜头与构图
→ 光影与色彩
→ 服装、材质、道具和环境细节
→ 风格实现
→ 任务相关限制
```

不要先写风格，再回头补主体。

## 输出形态判断

### 人物成片

重点关注：身份、外貌锚点、姿态、表情、服装、镜头和光影。

### 产品成片

重点关注：产品形态、展示角度、材质、结构、背景、光线、比例和文字准确性。

### 海报或封面

重点关注：视觉层级、主体位置、留白、标题区、色彩冲突和信息可读性。

### 电影或叙事画面

重点关注：事件发生中的瞬间、空间关系、角色视线、光源动机和情绪落点。

### 概念图

重点关注：设计目标、轮廓、功能、材质、尺度参照和世界观一致性。

## 输出骨架

```text
画面概述：
主要约束：
最终 Prompt：
备选 Prompt：
已自动补全项或方向决策摘要：
```

备选 Prompt 不是必须；只有确实存在另一条有价值方向时再提供。

## 常见失败与修复

- 主体不清楚：减少风格词，增加身份、轮廓和辨识锚点
- 场景漂亮但无叙事作用：明确主体与环境的关系
- 镜头与光影冲突：让光源方向服务当前机位和视觉重点
- 细节很多但主次不明：收窄背景信息，重新建立第一视觉落点
- 画面像模板图：补具体事件、动作阶段和环境证据
- 限制项过多：只压制当前最危险的失败模式
