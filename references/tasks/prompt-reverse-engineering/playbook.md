# 参考图反推 Prompt Playbook

## 用途

根据参考图和用户目标，生成复现型、结构型或改造型 Prompt。

## 何时读取

适用于：

- 根据图片反推出可复现的完整 Prompt
- 分析图片为什么有效，并提炼可复用结构
- 基于原图设计新的主体、场景或版本
- 从参考图提取镜头、构图、光影、色彩和材质方法

## 第一步：根据用户意图选路线

### 复现型

用户希望尽量生成与原图相似的画面。

重点：

- 主体身份和外观
- 场景与空间关系
- 构图、景别、机位和透视
- 动作、表情和视线
- 光源方向和色调
- 材质、画面风格和比例
- 容易漂移的限制项

### 结构型

用户希望提炼原图的可复用方法，而不是照抄内容。

重点：

- 视觉主次公式
- 主体与环境比例
- 镜头和构图模式
- 光影和配色逻辑
- 动作或叙事关系
- 可以替换的主体、场景和道具槽位

### 改造型

用户希望继承原图部分优势，但生成一个新的画面目标。

重点：

- 原图中可继承的内容
- 新目标必须覆盖的内容
- 不应继续继承的元素
- 主体、场景、风格和镜头的替换边界

用户意图明确时直接选择路线。只有“分析这张图，给我 Prompt”且用途无法判断时，才追问一个最关键问题。

## 第二步：拆解参考图

按以下顺序分析：

1. 主体和第一视觉落点
2. 场景和前中后景
3. 主体动作、姿态和接触关系
4. 景别、机位、焦段感和透视
5. 构图与视觉重量
6. 光源、阴影、色温和色彩结构
7. 材质、服装、妆容和道具
8. 风格实现方式
9. 画幅比例与裁切
10. 容易被模型误解的部分

## 复现型输出

```text
原图视觉拆解：
必须保留：
镜头与构图：
光影与色彩：
主体和场景细节：
完整复现 Prompt：
防漂移限制：
```

不要只堆风格词，必须写出画面关系。

## 结构型输出

```text
原图有效机制：
视觉结构公式：
主体替换槽位：
场景替换槽位：
镜头与光影公式：
通用 Prompt 模板：
适用范围与限制：
```

结构型结果应允许用户换人物、换场景、换题材后继续使用。

## 改造型输出

```text
原图可继承项：
新目标覆盖项：
禁止继承项：
改造后的视觉关系：
最终改造 Prompt：
限制项：
```

## 反推原则

- 描述可见画面，不编造无法从图中确认的品牌、人物身份和拍摄设备型号
- 可以判断焦段感、光线关系和视觉风格，但不把推测写成确定事实
- 参考图存在缺陷时，不应为了复现而无条件继承
- 图中风格与内容必须分开拆解，避免换主体后仍复制原人物特征
- 复现型追求相似，结构型追求可复用，改造型以新目标为最高优先级

## 常见失败与修复

- 只有图片复述，没有 Prompt：把分析转为可执行语义
- 反推结果过于笼统：补镜头、空间、动作、光影和材质关系
- 结构型仍然绑定原人物：改写为槽位和变量
- 改造型被原图绑死：明确新目标覆盖项和禁止继承项
- 把猜测当事实：使用“呈现为、具有、类似”的视觉描述
