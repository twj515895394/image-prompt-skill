# 角色资产 Playbook

## 用途

生成可复用、可延续、可用于多场景和多镜头的角色资产，而不是只做一张漂亮成片。

## 何时读取

适用于角色设定图、三视图、多角度头像、表情板、姿态板、换装板、手部鞋履饰品道具细节和角色主视觉。

## 核心目标

先锁定角色识别，再扩展可变资产。

```text
身份锚点
→ 外貌锚点
→ 轮廓与比例
→ 发型、服装和配色
→ 固定道具
→ 可变表情、姿态和换装
→ 特殊剧情状态
→ 一致性限制
```

## 固定项

至少判断：

- 年龄感、身份和常态气质；
- 脸型、眼距、眼型、鼻唇比例、下颌和耳位；
- 发际线、分线、发型主轮廓和发色；
- 头身比、肩胯关系、体型和四肢比例；
- 主服装轮廓、长度关系和核心配色；
- 标志性饰品、伤疤、痣或固定道具。

详细固定方法读取 `../../controls/identity-consistency/character-identity-anchors.md`。

## 可变项

- 表情和表情强度；
- 姿态、动作和重心；
- 换装与临时配饰；
- 场景和镜头；
- 局部妆面强度；
- 光影和色调；
- 战损、淋雨、疲惫等剧情状态。

固定项和可变项必须明确分开。

## 推荐资产包

按需要选择，不要求每次全部生成：

1. 中性背景角色识别主图；
2. 正、侧、背三视图；
3. 4–6 张多角度头像；
4. 基础表情组；
5. 常用姿态和习惯性小动作；
6. 全身中性站姿；
7. 手部、鞋履、饰品和固定道具细节；
8. 换装组；
9. 特殊状态或剧情状态；
10. 场景化角色主视觉。

## 推荐出图顺序

先验证中性主图，再做三视图和头像；身份稳定后才进入表情、姿态、换装和场景化主视觉。不要从强光影或强场景主视觉开始定义角色。

## 输出形态重点

### 三视图

锁定比例、轮廓、服装结构、正侧背对应关系和中性站姿。

### 多角度头像

锁定脸部稳定区、发际线、眼距、鼻唇关系和耳位；角度变化不能变脸。

### 表情板

明确稳定区、可变化区域和强度；优先克制或明显表情，不让每格都爆发。

### 换装板

保持人物身份、发型主锚点、体型和气质；只改变服装、材质、配色与配饰。

### 姿态板

保持骨架比例，补重心、习惯动作和角色性格；姿态变化不能改变体型。

### 细节资产

手部、鞋履、饰品、固定武器或工具必须与主图的结构和配色对应。

## 常见失败与修复

- 多角度变脸：补脸部稳定区，降低表情强度；
- 换装像换人：把人物身份和服装变量分开；
- 三视图比例不一致：固定头身比、肩胯和服装长度关系；
- 资产包只出脸：回补全身、手部、鞋履和道具；
- 角色依赖场景氛围：先在中性背景验证身份；
- 发型变化导致身份漂移：保留发际线、发色或其他强锚点；
- 特殊状态覆盖原身份：战损、湿发和妆面变化只能叠加，不能替换核心识别特征。
