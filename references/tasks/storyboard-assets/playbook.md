# 分镜资产 Playbook

## 用途

生成用于叙事、动作设计和后续视频制作的故事板资产，重点是镜头顺序、动作阶段、空间关系和跨镜承接。

## 何时读取

适用于：

- 故事板总板
- 逐镜关键帧
- 动作拆解板
- 控制型故事板
- 风格型故事板
- 角色、场景和道具连续性参考

## 先锁定

- 总镜头数或大致镜头规模
- 每个镜头承担的叙事任务
- 角色、场景和道具固定锚点
- 镜头顺序与跨镜承接
- 最终故事板的版式和信息密度

## 单镜头字段

```text
镜号与时段
镜头任务
景别与机位
主体和站位
动作阶段
视线和互动
场景与道具状态
光影与色彩
运镜或方向提示
与上一镜、下一镜的承接
```

静态故事板 Prompt 可以描述运镜意图，但不能把完整视频时序执行混入单张图任务。

## 动作拆解

复杂动作不要只写终态，应根据需要拆为：

```text
准备
→ 发力
→ 接触或转折
→ 高潮
→ 落地或结果
```

每格只承担一个清楚动作阶段，避免同一格出现多个互斥姿态。

## 跨镜连续性

持续检查：

- 人物身份、服装和左右方向
- 道具持有者和状态
- 场景区域与出入口
- 角色站位和视线
- 动作起止姿态
- 光线方向和时间状态

## 控制板与风格板

### 控制型故事板

优先表达：镜头、站位、动作、方向、节奏和连续性。

### 风格型故事板

优先表达：最终画风、光影、色彩、材质和角色表现。

两者都应保留对方的必要信息，但侧重点不同。

## 常见失败与修复

- 每格只是不同姿势，没有叙事推进：重写镜头任务
- 动作跳跃：补准备、转折和结果阶段
- 左右方向反复变化：锁定空间原点和运动方向
- 道具状态不连续：逐镜标记道具持有与变化
- 总板漂亮但不可执行：补镜号、景别、动作和承接信息
