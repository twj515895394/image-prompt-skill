# 场景资产 Playbook

## 用途

生成可被成片、故事板、游戏场景和图生视频持续复用的场景资产，而不是只做一张漂亮空景。

## 何时读取

适用于场景总览、空景、俯视站位图、道具关系、空间动线、镜头入口、多角度场景参考、游戏关卡概念、等距地图和后续视频使用的空间锚点。

## 核心结构

```text
地点与用途
→ 时间、天气和环境状态
→ 空间原点与方向
→ 空间分区
→ 前中后景
→ 出入口和动线
→ 主道具与副道具
→ 人物站位和视线
→ 镜头入口
→ 光影与色彩
→ 稳定锚点与允许变化项
```

## 场景卡必须回答

- 这是哪里，空间用来发生什么；
- 空间原点是什么，门窗和主要区域朝哪个方向；
- 主体从哪里进入、移动和离开；
- 哪些道具决定场景辨识度；
- 角色与道具的相对位置；
- 后续镜头可以从哪里拍；
- 哪些结构跨图必须保持不变；
- 哪些时间、天气、人物和临时道具允许变化。

## 推荐资产包

1. 场景总览图；
2. 无人物空景图；
3. 俯视空间站位与动线图；
4. 主道具和关键区域细节；
5. 2–4 个稳定镜头入口；
6. 人物 A / B 或群像调度示意；
7. 昼夜、天气或特殊状态变体。

按实际用途选择，不机械要求全部生成。

## 游戏场景资产交付

当场景用于关卡、模拟经营、地图、概念设计或 2D / 3D 转换时，先读取 `../../libraries/composition-shot/game-asset-presentation-types.md`，选择：

- 场景概念总览；
- 等距地图 / 建筑剖面；
- 场景与角色比例板；
- 2D / 3D 转换对照；
- 像素战斗或游戏截图式场景；
- 宣传 Key Visual。

游戏场景不能只回答“看起来如何”，还应回答：

- 玩家或角色从哪里进入；
- 核心活动区与目标点在哪里；
- 哪些道路、楼梯、平台和门形成路径；
- 功能区如何连接；
- 哪些物体可交互、可收集或可作为掩体；
- 场景地标如何帮助方向识别；
- 角色、建筑、载具和道具的尺度关系。

## 空间原点与方向

场景资产建立时先固定：

```text
空间原点
+ 场景前后左右
+ 出入口
+ 主活动区
+ 主道具位置
+ 角色常用起点与终点
```

详细站位、三角关系、视线和动线读取 `../../controls/spatial-blocking/staging-and-direction-control.md`。

## 空间一致性

### 固定

- 出入口方向；
- 主体活动区；
- 主道具位置；
- 空间尺度关系；
- 窗户、门、道路、舞台或功能区位置；
- 决定场景辨识度的材料、结构和色彩锚点。

### 允许变化

- 人物站位；
- 临时道具；
- 光线状态；
- 天气与时间；
- 机位和景别；
- 剧情造成的局部破损或使用痕迹。

变化不能破坏空间原点和核心结构。

## 镜头与玩家入口

至少考虑：

- 总览入口：交代空间和尺度；
- 关系入口：拍人物互动；
- 动作入口：完整看到运动路径；
- 细节入口：拍关键道具或局部事件；
- 玩家入口：说明进入空间后的第一个地标和可行路径；
- 目标入口：说明核心任务区或互动点如何被看见。

机位变化时，门窗、主道具和活动区仍需对应同一空间。

## 等距与像素场景

### 等距微缩世界

需要多层功能区、连接结构、统一尺度和高信息密度时，读取 `../../styles/3d-rendering/isometric-miniature-world.md`。

### 像素场景

需要 Sprite、Tile、复古战斗画面或像素环境时，读取 `../../styles/illustration/pixel-art-visual-language.md`，固定逻辑分辨率、像素尺度、色板和 Tile 边界。

## 出图顺序

```text
1. 场景总览
2. 无人物空景
3. 俯视站位、路径与功能区
4. 关键区域和主道具细节
5. 稳定镜头或玩家入口
6. 人物调度与角色尺度图
7. 昼夜和天气变体
8. 按需生成宣传 Key Visual
```

先验证空间一致，再增加人物、氛围变化和宣传视觉。

## 常见失败与修复

- 空景漂亮但无法表演：补活动区、动线、站位和镜头入口；
- 游戏场景漂亮但无法游玩：补玩家入口、目标点、路径和功能区；
- 多张场景图像不同地点：锁定空间原点、门窗、主道具和尺度；
- 道具很多但无主次：保留决定辨识度和功能的核心道具；
- 俯视图与透视图不一致：先定义场景方向和固定节点；
- 场景依赖单一机位：补总览、关系、动作、细节和玩家入口；
- 昼夜变体像不同场景：只改变光线和天气，不改核心结构；
- 等距场景区域无功能：为每区定义用途和连接关系；
- 人物进入后空间被重构：人物站位是变量，场景结构不是变量。
