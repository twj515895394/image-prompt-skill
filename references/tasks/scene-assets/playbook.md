# 场景资产 Playbook

## 用途

生成可被成片、故事板和图生视频持续复用的场景资产，而不是只做一张漂亮空景。

## 何时读取

适用于场景总览、空景、空间站位、道具关系、镜头入口、多角度场景参考和后续视频使用的空间锚点。

## 核心结构

```text
地点与用途
→ 时间、天气和环境状态
→ 空间分区
→ 前中后景
→ 出入口和动线
→ 主道具与副道具
→ 人物站位和视线
→ 镜头入口
→ 光影与色彩
→ 稳定锚点
```

## 场景卡必须回答

- 这是哪里，空间用来发生什么
- 主体从哪里进入、移动和离开
- 哪些道具决定场景辨识度
- 角色与道具的相对位置
- 后续镜头可以从哪里拍
- 哪些结构跨图必须保持不变

## 推荐资产包

1. 场景总览图
2. 无人物空景图
3. 俯视空间站位图
4. 主道具和关键区域细节
5. 2–4 个稳定镜头入口
6. 昼夜、天气或特殊状态变体

按实际用途选择，不机械要求全部生成。

## 空间一致性

固定：

- 出入口方向
- 主体活动区
- 主道具位置
- 空间尺度关系
- 窗户、门、道路、舞台或功能区位置

允许变化：

- 人物站位
- 临时道具
- 光线状态
- 天气与时间
- 机位和景别

## 常见失败与修复

- 空景漂亮但无法表演：补活动区、动线和站位
- 多张场景图像不同地点：锁定出入口、主道具和尺度关系
- 道具很多但无主次：保留决定辨识度的少量核心道具
- 俯视图与透视图不一致：先定义空间原点和方向
- 场景依赖单一机位：补可复用的镜头入口
