# 整体图生图 Playbook

## 用途

在保留关键参考锚点的前提下，对整张画面进行定向改写。

## 何时读取

适用于：

- 整体换风格
- 保留人物，更换场景
- 更换服装、妆容、光影或色调
- 改变构图、景别和观看关系
- 提升真实感、完成度或叙事性
- 基于原图进行较大幅度二次创作

只修改一个局部且其他内容必须不变时，改用 `image-editing`。

## 先判断改写强度

### 高保留

- 主体、构图和主要空间关系尽量接近
- 只做质感优化、错误修复或有限变化

### 中保留

- 保留主体身份和核心气质
- 允许改场景、镜头、服装、光影和色调

### 低保留

- 只借用主体锚点、局部结构或视觉方法
- 新目标优先于原图完整保留

强度由用户意图决定，不设置固定默认答案。

## 核心改写合同

```text
必须保留：
必须改写：
允许变化：
只借用：
禁止继承：
最终画面目标：
```

## 执行顺序

1. 确认参考图的主职责
2. 提取主体、场景、构图、光影和材质锚点
3. 明确保留与改写边界
4. 判断新目标会影响哪些视觉层
5. 重建主体、空间、镜头、光影和风格之间的关系
6. 补充与当前目标相关的限制项

## 允许共同变化的维度

- 场景
- 构图
- 景别和机位
- 服装、妆容和发型状态
- 光影、色彩和天气
- 动作、表情和情绪强度
- 材质与风格实现

但必须显式锁定用户要求保留的身份和关键锚点。

## 常见失败与修复

- 结果与原图几乎没差别：提高改写层级，明确镜头、场景或光影变化
- 主体身份漂移：减少外貌变化，补身份锚点
- 新场景像贴图：重写人物与环境的光影、透视和接触关系
- 只换风格不换画面：补构图、动作和空间改写目标
- 所有参考都被平均融合：重新分配主职责和只借用项
