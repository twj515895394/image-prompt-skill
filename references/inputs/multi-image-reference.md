# 多张参考图分析

## 用途

为多张参考图分配清晰职责，处理冲突和优先级，避免把所有图片平均融合。

## 何时读取

用户提供两张或以上图片，并希望组合人物、服装、场景、构图、动作、光影、风格或局部元素时读取。

## 第一步：识别用户意图

先判断用户是在做：

- 同一人物的一致性延续
- 人物与服装组合
- 人物与场景组合
- 多人物分别绑定参考
- 构图、姿态或动作借用
- 光影或风格迁移
- 局部物体替换
- 多图整合后的新创作

## 第二步：为每张图分配职责

可用职责包括：

- 人物 A 身份
- 人物 B 身份
- 脸部与五官
- 体型与比例
- 发型
- 妆容
- 服装
- 姿态与动作
- 场景与空间
- 道具
- 构图与镜头
- 光影与色彩
- 材质
- 风格
- 局部细节
- 负面参考：明确哪些错误不能继承

不要用“主图负责全部、辅图只补细节”作为固定模板。主图与辅助图的划分必须服务用户当前目标。

## 第三步：绑定对象

多人物任务必须明确：

```text
人物 A ← 哪张图提供身份、服装和姿态
人物 B ← 哪张图提供身份、服装和姿态
场景 ← 哪张图提供空间
构图 ← 哪张图提供镜头关系
风格 ← 哪张图只提供视觉表现
```

不得把人物 A 的脸、人物 B 的服装和第三张图的姿态无意混合到同一个角色上。

## 第四步：处理同维度冲突

同一维度由多张图提供不同答案时，按以下方式处理：

1. 用户明确指定者优先
2. 与当前任务目标最相关者优先
3. 身份参考与风格参考分离
4. 不能同时成立的要求必须做取舍
5. 无法判断且会显著改变结果时，只追问一个最关键问题

## 第五步：确定保留强度

每张图分别标记：

- 必须精确保留
- 保留主要特征
- 只借用局部
- 只借用风格或方法
- 不继承图中其他内容

## 输出草稿

```text
图 1 职责：
图 2 职责：
图 3 职责：
对象绑定：
必须保留：
必须修改：
允许融合：
禁止混合：
冲突处理：
最终参考优先级：
```

## 常见失败与修复

- 平均融合导致谁都不像：重新分配单一职责
- 人物串脸或串衣服：补对象绑定
- 风格图改变人物身份：明确风格图不提供身份
- 场景图污染人物外观：限制场景图只提供空间和光影
- 多图要求互斥：按用户目标确定主优先级，不强行兼容
