# 纯文字输入扩写

## 用途

把用户的文字创意补成可执行画面结构，但不擅自改变题材、世界观和关键人物关系。

## 何时读取

用户没有提供参考图，只通过文字描述生图、资产或参考帧需求时读取。

## 核心判断

先识别：

1. 当前任务是什么
2. 主体是谁或是什么
3. 画面要表达什么事件、状态或情绪
4. 场景落在哪里
5. 哪些内容用户已经锁定
6. 哪些内容允许自动补全

## 扩写顺序

```text
主体
→ 场景
→ 动作或状态
→ 空间关系
→ 镜头与构图
→ 光影与色彩
→ 服装、材质或道具
→ 风格
→ 限制项
```

## 补全优先级

### 不补就会跑偏

- 主体身份
- 场景地点
- 当前任务目标
- 人物或物体数量

### 影响表现力

- 景别和机位
- 动作阶段
- 光源方向
- 主色与冷暖关系

### 影响完成度

- 材质细节
- 环境层次
- 局部道具
- 任务相关限制项

## 不主动补全

- 未被暗示的强剧情
- 会改变题材的大方向
- 用户未要求的世界观设定
- 关键人物关系
- 会把单图任务扩展成长篇叙事的内容

## 任务联动

完成输入扩写后，必须继续读取当前任务 Playbook。输入层不直接决定资产包、编辑合同或最终输出结构。

## 输出草稿

```text
已知主体：
已知场景：
已知动作或状态：
必须保留：
允许补全：
需要任务 Playbook 决定的内容：
```
