# 单张参考图分析

## 用途

把一张参考图拆成当前任务可使用的视觉锚点，并明确哪些内容保留、修改、借用或禁止继承。

## 何时读取

用户提供一张图片，用于复现、改写、编辑、资产生成、风格借用或 Prompt 反推时读取。

## 先判参考图职责

- 成片参考
- 人物身份参考
- 场景参考
- 服装或道具参考
- 姿态与动作参考
- 构图与镜头参考
- 光影与色彩参考
- 风格参考
- 首帧、尾帧或关键帧参考

一张图可以承担多个职责，但必须区分主职责与辅助职责。

## 分析顺序

1. 用户想用这张图做什么
2. 图中哪些元素决定主体辨识度
3. 观众第一眼看哪里
4. 景别、机位、透视和构图关系
5. 光源方向、阴影、色调和材质
6. 哪些内容必须保留
7. 哪些内容必须修改
8. 哪些内容只借用方法或感觉
9. 哪些缺陷和无关内容不能继承

## 参考强度

根据用户意图判断：

- 高保留：主体、构图和关键视觉关系尽量接近
- 中保留：保主体与气质，允许改镜头、场景、服装或光影
- 低保留：只借用局部结构、风格或某个元素

不要默认所有参考图都走轻再创作；以用户当前目标为准。

## 冲突优先级

```text
用户当前文字要求
→ 用户明确指定的必须保留项
→ 参考图主职责
→ 参考图辅助信息
→ 自动补全
```

## 输出草稿

```text
参考图主职责：
必须保留：
必须修改：
允许变化：
只借用：
禁止继承：
镜头与构图要点：
光影与材质要点：
```

## 常见失败与修复

- 什么都保留：回到当前任务目标，删除无关锚点
- 只复述图片：补“如何用于当前任务”的转换关系
- 改写后主体漂移：增加身份和关键外观锚点
- 只有风格变化：补构图、场景、动作或光影的明确改写目标
