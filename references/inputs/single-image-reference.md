# 单张参考图分析

## 用途

把一张参考图拆成当前任务可使用的视觉锚点，并明确哪些内容保留、修改、借用或禁止继承。

## 何时读取

用户提供一张图片，用于复现、改写、编辑、资产生成、风格借用或 Prompt 反推时读取。

## 先判参考图职责

- 成片参考；
- 人物身份参考；
- 场景参考；
- 服装或道具参考；
- 姿态与动作参考；
- 构图与镜头参考；
- 光影与色彩参考；
- 风格参考；
- 首帧、尾帧或关键帧参考。

一张图可以承担多个职责，但必须区分主职责与辅助职责。

## 分析顺序

1. 用户想用这张图做什么；
2. 图中哪些元素决定主体辨识度；
3. 观众第一眼看哪里；
4. 场景的前中后景和主体位置；
5. 景别、机位、焦段感、透视和裁切；
6. 主体动作、姿态、接触和视线关系；
7. 光源方向、阴影软硬、色温和主体分离；
8. 皮肤、布料、金属、玻璃、颗粒等材质表现；
9. 风格由哪些线条、渲染、色彩或摄影变量实现；
10. 哪些内容必须保留、修改、只借用或禁止继承。

## 风格提取原则

当用户只借用参考图风格时，优先提取：

```text
色彩结构
+ 构图关系
+ 镜头与透视
+ 光影方向
+ 材质表现
+ 媒介或渲染方法
```

不要把原人物、原服装、原场景和原道具误当成风格的一部分。

风格提取后应转换为当前任务可执行的关系，而不是只给“复古、电影感、韩漫”等标签。

## 参考强度

根据用户意图判断：

- 高保留：主体、构图和关键视觉关系尽量接近；
- 中保留：保主体与气质，允许改镜头、场景、服装或光影；
- 低保留：只借用局部结构、风格或某个元素。

不要默认所有参考图都走轻再创作；以用户当前目标为准。

## 视频参考帧附加判断

如果参考图用于视频首帧、尾帧或关键帧，还要识别：

- 哪些元素可以运动；
- 哪些身份、服装、场景和道具必须固定；
- 当前动作处于起势、过程还是落点；
- 镜头后续可能向哪个方向运动；
- 哪些构图边缘需要给运动留空间。

输入层只提取这些锚点，具体帧结构仍由视频参考帧 Playbook 决定。

## 冲突优先级

```text
用户当前文字要求
→ 用户明确指定的必须保留项
→ 参考图主职责
→ 参考图辅助信息
→ 自动补全
```

## 输出草稿

```text
参考图主职责：
辅助职责：
必须保留：
必须修改：
允许变化：
只借用：
禁止继承：
第一视觉落点：
空间与构图要点：
镜头与透视要点：
光影与色彩要点：
材质与风格实现：
视频用途附加锚点：
```

无视频用途时省略最后一项。

## 常见失败与修复

- 什么都保留：回到当前任务目标，删除无关锚点；
- 只复述图片：补“如何用于当前任务”的转换关系；
- 改写后主体漂移：增加身份和关键外观锚点；
- 只有风格变化：补构图、场景、动作或光影的明确改写目标；
- 风格与内容混在一起：分别列出风格实现和可替换主体；
- 猜测设备或身份：只描述可见结果，不把推测写成事实。
