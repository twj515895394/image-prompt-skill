# 产品展示类型选择器

## 解决什么问题

根据产品结构、销售目标和画面用途，选择合适的展示方式，而不是默认把所有产品都居中放在纯色背景上。

## 什么时候读取

- 电商主图、产品广告、品牌视觉或产品系列图；
- 需要选择白底、场景化、悬浮、台座、组合陈列、爆炸图或包装展示；
- 产品、包装、配件和内容物需要明确同框关系；
- 用户只说“做一张高级产品图”，但没有展示类型。

## 展示类型

| 类型 | 适用目标 | 核心结构 | 风险 |
|---|---|---|---|
| 单品白底 | 准确展示形态、电商识别 | 单一主体、干净背景、轻阴影 | 过于平、缺少体积 |
| 单品极简背景 | 品牌感和商业主视觉 | 单品、渐变或纯色背景、少量留白 | 背景颜色影响产品主色 |
| 台座展示 | 美妆、珠宝、科技和奢侈品 | 产品与石材、玻璃或几何台座 | 台座抢主体或接触不自然 |
| 悬浮展示 | 科技、饮料、潮流商品 | 产品离地、软阴影或环境反射 | 漂浮无依据、重心不稳 |
| 对角动态展示 | 饮料、运动、潮流和能量型产品 | 主体倾斜、配料或碎片沿方向运动 | 产品形态被动态元素遮挡 |
| 场景化使用 | 家居、户外、食品和生活方式 | 产品处于真实使用环境 | 场景抢主体或产品太小 |
| 包装与内容物 | 食品、饮料、礼盒和收藏品 | 包装、实物、配件和内容物分层 | 数量和比例漂移 |
| 组合陈列 | 套装、系列、颜色矩阵 | 主次产品、前后层次和间距 | 平均排列、缺少主产品 |
| 平铺 Flat Lay | 服装、饰品、文具和食材 | 俯视、秩序排列、背景材质 | 物体重叠或透视错误 |
| 爆炸图 | 科技、机械和结构说明 | 部件沿单一轴线分层展开 | 层数混乱、零件缺失 |
| 剖面 / 切面 | 食品、包装和内部结构 | 外形与内部层级同时可读 | 内部结构不可信 |
| 手办 / 收藏品展示 | 玩具、模型、IP 周边 | 本体、底座、支撑件和包装关系 | 支撑件数量错误、比例漂移 |
| 食品主视觉 | 菜品、零食、饮料和甜点 | 食物主体、配料证据、质感和留白 | 食物变形、过度油亮 |

## 选择原则

```text
产品识别优先
→ 销售目标
→ 展示类型
→ 主体数量
→ 背景和台座
→ 配件 / 动态元素
→ 文字留白
```

- 形态准确优先于创意效果；
- 主产品必须有明确第一视觉权重；
- 套装图先确定主产品，再安排辅助产品；
- 动态元素只解释口味、能量、功能或结构；
- 爆炸图和剖面图必须说明部件数量、顺序和轴线；
- 包装和内容物同框时，明确哪些是实物、哪些是包装图案。

## Prompt 结构

```text
产品采用 [展示类型]，
主产品位于 [位置与角度]，占画面约 [比例]，
辅助产品 / 包装 / 配件位于 [关系]，
使用 [背景或台座] 建立层次，
保留 [品牌或文案安全区]，
禁止改变 [产品轮廓、比例、标签区域和配件数量]。
```

## 常见失败与修复

- 所有产品都居中白底：根据销售目标重新选择展示类型；
- 配件数量漂移：明确 exactly 数量和相对位置；
- 包装与内容物混淆：分别描述包装、实物和印刷图案；
- 场景化图产品太小：提高主体占比并降低背景复杂度；
- 爆炸图零件乱飞：固定单一分解轴线和层级顺序；
- 食品配料遮挡主体：把配料降为证据层，不做第一主体。
