# 海报与封面版式类型选择器

## 解决什么问题

根据主体、文字量、平台比例和阅读距离，选择合适的海报、封面、Banner 或缩略图版式，避免所有图文设计都变成主体居中加几行字。

## 什么时候读取

- 海报、宣传图、活动主视觉、社交媒体封面、YouTube 缩略图；
- 产品、人物、电影或品牌视觉需要预留标题和卖点区；
- 用户只说“做一张高级海报”，但没有版式；
- 画面主体和文字互相遮挡；
- 不同平台比例需要重新安排信息结构。

## 常用版式

| 版式 | 适用场景 | 核心结构 | 风险 |
|---|---|---|---|
| 主体居中 + 上下文字 | 活动、产品、电影海报 | 中央主视觉，上下标题与信息 | 容易模板化，主体与标题争抢 |
| 主体偏侧 + 对侧文字 | 人物、产品、品牌宣传 | 一侧主视觉，一侧文字安全区 | 两侧权重失衡 |
| 大标题压图 | 潮流、音乐、时尚、体育 | 标题成为主要图形，图像穿插其中 | 文字难读或遮挡主体 |
| 上图下文 | 教育、产品、活动说明 | 上部主视觉，下部信息区 | 下部信息过密 |
| 左图右文 | 产品、课程、人物访谈 | 图文双栏，阅读方向清晰 | 过于像网页卡片 |
| 对角动势 | 运动、饮料、游戏、科技 | 主体和文字沿对角线组织 | 动势太强导致信息不稳 |
| 留白极简 | 奢侈品、艺术展、品牌 | 少量文字、大面积负空间 | 缩小后信息不足 |
| 满版背景 + 信息块 | 电影、旅游、活动 | 背景占满，局部半透明或纯色文字块 | 背景纹理干扰文字 |
| 分栏网格 | 品牌提案、多个卖点、系列产品 | 规则网格和模块分区 | 模块平均化，没有主次 |
| 卡片矩阵 | 信息图、品牌板、社交长图 | 多个独立卡片和统一间距 | 卡片太多、阅读顺序混乱 |
| 缩略图高压缩 | 视频封面、移动端内容 | 大主体、短标题、高对比 | 小字过多、背景太复杂 |
| 纵向长图 | 社交平台、教程、步骤说明 | 自上而下分段阅读 | 首屏没有重点、段落过长 |

## 选择顺序

```text
平台比例
→ 第一视觉落点
→ 文字量
→ 主体数量
→ 阅读距离
→ 版式类型
→ 留白和安全区
```

## 缩略图规则

缩略图不是普通海报缩小版，应额外控制：

- 一个主要人物、产品或事件；
- 一个强情绪、冲突或卖点；
- 极短标题，通常只保留核心词；
- 强明度或色彩对比；
- 移动端缩小后仍可读；
- 不依赖小字、复杂背景和多层说明；
- 标题不遮挡五官、产品标签或关键动作。

## 平台比例

- 16:9：视频缩略图、横版 Banner、电影感宣传；
- 4:5：社交媒体人物或产品宣传；
- 1:1：头像、社交卡片、单主体促销；
- 9:16：移动端封面、短视频竖版海报；
- A4 / 2:3 竖版：传统活动海报和印刷宣传；
- 超长竖版：教程、流程和社交长图。

比例变化后应重新排版，不要简单裁切。

## Prompt 结构

```text
采用 [版式类型]，适配 [比例与平台]，
第一视觉落点为 [主体]，位于 [位置]，
标题区位于 [位置]，副标题和辅助信息按 [阅读顺序] 排列，
在 [位置] 保留稳定负空间，
图像、文字和装饰元素保持 [主次关系]，
缩小到 [移动端 / 缩略图] 后仍能识别主体和核心标题。
```

## 常见失败与修复

- 所有海报都主体居中：根据文字量和平台重新选择版式；
- 标题压住主体：建立标题安全区和遮挡边界；
- 缩略图小字太多：只保留核心词和一个主视觉；
- 模块平均无重点：确定主模块并通过尺寸、位置或对比强化；
- 竖版直接裁横版：重新安排主体、文字和留白；
- 背景纹理影响文字：在文字区降低细节或增加稳定信息块。
