# 故事板类型与版式选择器

## 解决什么问题

根据故事板用途选择合适的总板类型、信息密度和版式，避免所有分镜都使用同一种九宫格，也避免风格板漂亮但不可执行，或控制板信息很多却无法预览最终画面。

## 什么时候读取

- 用户明确要求故事板总板、动作拆解板、控制板或风格板；
- 需要决定每格承载多少镜头信息；
- 需要在角色、场景、动作、镜头和最终画风之间选择重点；
- 用户要求 `image2` 故事板、分镜参考图或一张总板；
- 当前故事板布局不适合镜头数量或制作阶段。

单个镜头如何设计由 storyboard Playbook 和 camera controls 决定；本页只提供可选择的总板形态。

## 常用故事板类型

| 类型 | 主要用途 | 重点信息 | 常见风险 |
|---|---|---|---|
| 叙事顺序板 | 展示完整剧情推进 | 镜号、事件、动作、承接 | 每格像独立插画，没有推进 |
| 控制型故事板 | 控制镜头和后续视频执行 | 景别、机位、站位、方向、动作阶段、节奏 | 画面过度符号化，缺少角色和场景识别 |
| 风格型故事板 | 预览最终画风和质感 | 画风、光影、色彩、材质、表演 | 漂亮但镜头、动作和连续性不足 |
| 动作拆解板 | 拆复杂动作或运动过程 | 准备、发力、转折、高潮、结果 | 同一格塞入多个动作阶段 |
| 角色表演板 | 展示表情、姿态和关系变化 | 眼神、重心、手势、互动、情绪节拍 | 角色身份和身体比例漂移 |
| 场景调度板 | 规划站位、动线和镜头入口 | 空间原点、出入口、人物路径、道具位置 | 俯视图与透视镜头不一致 |
| 关键帧板 | 为视频首帧、尾帧和过渡帧提供参考 | 起点、转折、落点、可运动元素 | 被误当作完整视频时间轴 |

## 控制板与风格板的关系

两者不是互斥交付：

### 控制型优先

- 镜号和顺序；
- 站位、动作和方向；
- 景别、机位和构图；
- 道具与连续性；
- 节奏和跨镜承接。

同时保留最低风格信息：角色外观、场景识别、基础光色。

### 风格型优先

- 最终画风；
- 光影、色彩和材质；
- 角色表现和场景气氛；
- 成片完成度。

同时保留最低控制信息：镜号、核心动作、站位、视线和承接。

## 按镜头数量选择版式

| 镜头数量 | 推荐布局 | 使用说明 |
|---|---|---|
| 1–4 镜 | 1×2、2×2 或大格混排 | 允许保留更多角色和场景细节 |
| 5–6 镜 | 2×3、3×2 | 适合短动作、短广告和单场景关系戏 |
| 7–9 镜 | 3×3 或主次混排 | 适合一个完整短段落 |
| 10–12 镜 | 3×4、4×3 | 需要严格控制每格信息密度 |
| 超过 12 镜 | 拆分多个 Pack | 不把所有镜头压进一张不可读总板 |

版式应服从镜头数量和阅读顺序，不为了对称强行压缩。

## 单格最低信息

总板每格至少可识别：

- 镜号；
- 主体；
- 核心动作或状态；
- 景别或观看关系；
- 与上一格的变化。

控制型板可增加：

- 箭头；
- 运动方向；
- 站位标记；
- 焦段或机位；
- 道具状态；
- 时段。

## 底部控制轨道

需要统一管理全片时，可在总板底部增加控制轨道，记录：

- 时间轴；
- 镜头编号；
- 动作节拍；
- 情绪曲线；
- 道具状态；
- 角色和场景固定项；
- 跨 Pack 衔接。

控制轨道不应挤压主要画面到不可读。

## Prompt 结构

```text
生成一张 [故事板类型]，覆盖 [镜头数] 个镜头，
采用 [行 × 列 / 主次混排] 布局，按 [阅读顺序] 排列。
每格清楚展示 [主体、动作、景别和承接]。
重点强化 [控制信息 / 风格信息 / 动作阶段 / 空间调度]，
同时保留 [另一类必要信息]。
底部 [需要 / 不需要] 控制轨道，包含 [字段]。
```

## 常见失败与修复

- 所有任务都用九宫格：按镜头数和用途重新选择版式；
- 总板漂亮但不可执行：补镜号、动作、方向和承接；
- 控制板像工程草图无法识别角色：保留角色外观和场景锚点；
- 风格板每格画风不一致：统一风格、光色和材质规则；
- 超过 12 镜仍压一张：拆分 Pack，并写清跨 Pack 衔接；
- 每格信息过多：每格只保留一个核心动作或叙事任务。
