# 游戏资产交付类型选择器

## 解决什么问题

根据游戏制作、宣传或资产验证目标选择正确的交付形态，避免把所有“游戏资产”都做成一张角色海报。

## 什么时候读取

- 用户要求游戏角色、场景、道具、图标、精灵图、贴纸表、资产表或宣传 Key Visual；
- 需要判断角色设定板、场景概念图、图标矩阵、像素战斗画面或 2D / 3D 对照表；
- 同一角色或场景要进入后续动画、建模、UI 或游戏开发；
- 用户只说“做一个游戏资产”，但没有说明交付用途。

## 交付类型

| 类型 | 适用目标 | 核心内容 | 风险 |
|---|---|---|---|
| 宣传 Key Visual | 商店封面、活动宣传、角色登场 | 英雄主体、世界观背景、品牌留白 | 只有气氛，没有资产可复用性 |
| 角色设定板 | 角色设计、动画、建模 | 主图、三视图、表情、姿态、细节 | 信息过多导致身份漂移 |
| 剪影探索板 | 早期造型方向 | 多个黑白或低细节轮廓 | 只变姿势，不变设计方向 |
| 表情 / 贴纸表 | 表情包、UI 头像、社交资产 | 规则网格、统一比例、精确数量 | 每格变脸或尺寸不一 |
| 精灵图 Sprite Sheet | 2D 游戏动画 | 固定画布、帧序、动作循环 | 帧尺寸、基线和朝向漂移 |
| 像素战斗画面 | 游戏截图、战斗演示 | 角色、场景、UI 或技能效果 | 像素尺度混乱、特效遮挡角色 |
| 道具 / 图标矩阵 | 背包、技能、资源和装备 | 单元格、统一角度、统一背景 | 图标风格和光向不一致 |
| 场景概念总览 | 关卡、地图和世界观 | 空间功能、动线、地标和尺度 | 漂成漂亮空景，无法游玩 |
| 等距地图 / 剖面 | 模拟经营、基地、功能空间 | 功能区、连接结构、道具和路径 | 密度高但不可读 |
| 2D / 3D 转换对照 | 建模验证、视觉开发 | 参考图、转换结果、结构说明 | 两侧不是同一对象 |
| 载具 / 武器资产板 | 工业设计和战斗系统 | 侧视、三视、展开、材质和细节 | 随机新增部件 |
| 场景与角色比例板 | 关卡和镜头尺度 | 角色、建筑、载具和道具比例 | 比例只凭视觉感觉，无统一基准 |

## 选择顺序

```text
下游用途
→ 单张展示或系列资产
→ 角色 / 场景 / 道具对象
→ 是否需要多角度或多状态
→ 是否需要透明背景、网格或标签
→ 最终交付类型
```

## Key Visual 与资产板的区别

- Key Visual 优先视觉冲击、世界观和宣传完成度；
- 资产板优先结构、身份、比例和可复用信息；
- Key Visual 可以使用强透视、特效和环境；
- 资产板应使用更稳定角度、背景和光线；
- 不要用一张强气氛海报替代三视图、精灵表或图标矩阵。

## 网格型资产要求

表情、图标、贴纸和精灵表应固定：

- 行列数；
- 单元格尺寸；
- 角色或对象占比；
- 背景规则；
- 朝向和基线；
- 精确数量；
- 是否允许文字标签；
- 单元格之间的安全间距。

## Prompt 结构

```text
目标是生成 [游戏资产交付类型]，用于 [下游用途]。
主对象为 [角色 / 场景 / 道具 / 载具]，
画布采用 [比例、网格或分栏]，
必须包含 [具体模块与精确数量]，
统一 [身份、比例、角度、光向、背景和渲染方式]，
禁止 [额外对象、数量漂移、结构改造和跨格风格变化]。
```

## 常见失败与修复

- 游戏资产被做成普通海报：重新确认下游用途和交付类型；
- 表情表数量错误：固定行列、单元格和 exactly 数量；
- 精灵帧不连续：固定基线、朝向、动作输入输出和帧尺寸；
- 图标矩阵风格漂移：统一角度、背景、光向和材质；
- 场景概念图无法用于关卡：补功能区、路径、地标和角色尺度；
- 2D / 3D 对照不是同一对象：重复身份、轮廓、配色和关键结构锚点。
