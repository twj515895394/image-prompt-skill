# 角色发型选择器

## 定位

把“黑长直”“短发”“卷发”等粗粒度描述，细化为可用于角色一致性和 Prompt 组装的发型锚点。

## 何时读取

- 发型是角色辨识度的重要部分；
- 用户只给出粗粒度发型词，需要细化；
- 角色资产需要跨角度、跨表情和跨场景保持一致；
- 妆容、职业、时代或题材需要与发型协同；
- 动作镜头需要控制发丝动态和面部遮挡。

## 选择顺序

```text
角色身份与气质
→ 基础长度
→ 发型形态
→ 刘海或分线
→ 发色
→ 配饰
→ 发丝质感和动态
```

## 发型锚点字段

- 长度；
- 形态；
- 刘海或分线；
- 发色；
- 配饰；
- 发丝质感；
- 动态状态。

角色一致性任务中至少固定前五项。跨镜头不要随意改变长度、刘海、分线和发色。

## 镜头与动态联动

- 长发、大波浪和云朵卷适合风吹、回头、奔跑和慢镜头；
- 高马尾、双马尾和丝带辫适合青春感和动作感；
- 低马尾、低盘发和短发 BOB 适合职场对白和冷静角色；
- 湿发、凌乱发丝适合雨夜、战损、崩溃和暧昧状态；
- 近景和侧颜画面必须明确发丝是否遮挡眼睛、鼻梁或下颌线；
- 强风和高速运动会增加一致性风险，不应在所有镜头中默认使用。

## 组合模板

```text
[角色身份]，[基础长度]，[发型形态]，
[刘海或分线]，[发色]，[配饰]，
发丝呈现 [质感与动态]，
整体服务 [角色身份与气质]，
与 [妆容、服装和场景] 保持一致。
```

## 避坑

- 只写黑长直，不写分线、刘海、长度和发丝状态；
- 每张图随机改变刘海、分线或发色；
- 动作画面中发丝完全静止；
- 发型与职业、时代、妆容和人物气质冲突；
- 动态发丝遮住关键五官，却没有明确遮挡范围；
- 为追求变化频繁换发型，破坏角色识别。
