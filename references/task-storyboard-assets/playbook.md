# Storyboard Assets Playbook

## 目标

生成逐镜图、动作拆解图和故事版 Prompt，让后续视频阶段有稳定的镜间承接。

## 先判故事版类型

- 逐镜故事版
- 角色故事版
- 场景固定故事版
- 场景规划故事版
- 动作拆解故事版
- 动作设计故事版
- 场景概览故事版

## 核心结构

1. 先写镜号字段
2. 再写这一镜的唯一核心任务
3. 再写动作阶段
4. 再写镜间承接
5. 最后补光影、风格和限制

## 每一镜必写字段

- 镜号
- 画面目标
- 主体
- 场景
- 景别与机位
- 角色动作
- 情绪
- 光影色调
- 可运动元素
- 承接上一镜
- 引出下一镜

## 推荐输出包

1. 镜头列表图
2. 逐镜关键帧
3. 动作拆解图
4. 场景站位图
5. 首帧与尾帧标记图

## 输出骨架

```text
故事版类型：
镜号：
本镜任务：
主体与场景：
景别/机位：
动作阶段：
承接上一镜：
引出下一镜：
分镜资产 Prompt：
```

## 使用原则

- 每一镜只承担一个核心变化
- 复杂动作先拆阶段，再美化
- 分镜图的第一价值是关系和动作可读，不是成片质感

## 常见失败

- 一张分镜图里塞太多动作
- 镜号与承接关系缺失
- 只写氛围，不写镜头任务

## 修复方式

- 读不清主任务：把一镜拆成两镜
- 镜间断裂：补“承接上一镜 / 引出下一镜”
- 动作混乱：回到起势、过程、落点三段式
