# 像素艺术视觉语言

## 解决什么问题

定义真正基于像素网格、有限色板和像素簇组织的视觉语言，避免把普通插画加马赛克或模糊滤镜误当成像素艺术。

## 什么时候读取

- 用户明确要求像素画、复古游戏画面、Sprite、像素场景、像素图标或像素战斗画面；
- 角色和场景需要在低分辨率下保持可读；
- 画面出现像素大小混乱、局部抗锯齿或高清纹理混入；
- 需要区分像素艺术、体素 3D 和普通低清插画。

## 核心公式

```text
统一逻辑分辨率
+ 清楚像素簇
+ 有限色板
+ 无模糊抗锯齿
+ 小尺寸可读轮廓
+ 受控明暗与抖动
```

## 第一步：确定用途

| 用途 | 核心要求 |
|---|---|
| 角色 Sprite | 轮廓、朝向、动作基线和帧尺寸一致 |
| 战斗截图 | 角色、敌人、技能效果和场景层级清楚 |
| 环境场景 | 建筑、地形、前中后景和光源可读 |
| 图标 / 道具 | 单元格一致、轮廓简洁、背景规则统一 |
| 像素插画 | 允许更高细节，但仍保持统一像素尺度 |
| Tile / 地图块 | 边缘可拼接、透视和光向固定 |

## 逻辑分辨率与像素尺度

- 先决定低、中或高细节像素尺度；
- 同一张图中的角色、背景和特效使用一致像素大小；
- 不允许脸部是细小像素，背景却是大方块；
- 放大展示应使用整数倍最近邻放大感；
- 避免柔化、景深模糊、抗锯齿和照片纹理。

## 像素簇与边缘

- 轮廓由有意组织的阶梯边缘构成；
- 大面积明暗使用像素簇，不使用连续渐变；
- 曲线通过稳定阶梯节奏表达；
- 单个孤立噪点数量受控；
- 斜线、圆形和武器边缘需要统一像素节奏；
- 角色五官在小尺寸下优先识别，不追求微小写实细节。

## 色板

- 使用有限主色、阴影色、高光色和强调色；
- 同一材质共享稳定色阶；
- 夜景通过色相偏移和明度层次表达，不只整体压黑；
- 技能和粒子效果可以使用强调色，但不能覆盖角色轮廓；
- 复古感来自色板和像素结构，不依赖具体旧游戏名称。

## 明暗、抖动与材质

- 光源方向统一；
- 阴影使用块面或有限色阶；
- Dithering 只用于色阶过渡、空气或特殊材质；
- 不在所有区域均匀铺抖动纹理；
- 金属、石材、水面和布料通过不同像素簇和高光形状区分；
- 水面反射和火焰需要保持像素尺度，不变成高清粒子。

## Sprite 与动画帧

固定：

- 画布尺寸；
- 脚底基线；
- 朝向；
- 角色占比；
- 轮廓和色板；
- 动作帧顺序；
- 每帧输入与输出姿态。

动作循环应包含准备、主动作、接触或峰值、恢复，不要每帧只是随机姿势。

## Prompt 结构

```text
采用真正的像素艺术视觉语言，目标用途为 [Sprite / 战斗截图 / 环境 / 图标 / 插画]，
使用统一 [逻辑分辨率与像素尺度]，
角色和场景由清楚像素簇与阶梯边缘构成，
色板限制为 [主色、阴影、高光和强调色关系]，
光源来自 [方向]，使用块面阴影和受控 dithering，
禁止抗锯齿、照片纹理、柔焦、混合像素尺寸和局部高清渲染。
```

## 与相邻风格的边界

- 体素 3D 使用立方体几何、真实空间光照和三维体积，不等于 2D 像素画；
- 低多边形强调面数和几何切面，不等于像素网格；
- 普通插画缩小后加噪点，不构成像素艺术；
- 需要游戏交付格式时联动游戏资产类型选择器。

## 常见失败与修复

- 像普通插画加马赛克：重建逻辑分辨率、像素簇和有限色板；
- 像素尺寸忽大忽小：固定全画面像素尺度；
- 边缘模糊：移除抗锯齿、柔焦和景深；
- 技能特效盖住角色：降低面积和亮度，恢复角色轮廓；
- Sprite 每帧变形：固定基线、占比、身份和动作输入输出；
- 夜景一片黑：增加有限色阶和色相偏移层次；
- 抖动纹理过多：只在需要过渡和材质的局部使用。
