# 信息图视觉语言

## 解决什么问题

把复杂信息转成与其逻辑关系匹配的视觉结构，避免信息图只是图标、卡片和文字的平均堆叠。

## 什么时候读取

- 教育信息图、流程图、时间线、对比图、数据卡片和结构示意图；
- 品牌提案板、产品卖点板或社交平台长图；
- 用户要求“清晰、专业、容易理解”的图文解释；
- 多个模块之间缺少阅读顺序或因果关系；
- 画面需要同时容纳图标、数据、短文字和示意图。

## 第一步：识别信息关系

| 信息关系 | 推荐结构 | 典型用途 |
|---|---|---|
| 流程 | 箭头、步骤链、泳道或节点路径 | 操作步骤、业务流程、制作过程 |
| 时间 | 水平或垂直时间线 | 历史、项目进度、生命周期 |
| 对比 | 左右双栏、矩阵或前后对照 | 产品、方案、状态、优缺点 |
| 层级 | 金字塔、树状、同心层或缩进结构 | 组织、能力、优先级、分类 |
| 分类 | 分组卡片、象限或标签簇 | 主题归纳、用户类型、功能分类 |
| 因果 | 起因—机制—结果、鱼骨或关系箭头 | 问题分析、风险、影响链路 |
| 组成 | 爆炸图、剖面、环形分区或局部标注 | 产品结构、系统组成、人体或设备 |
| 地理 | 地图、路线、区域高亮 | 旅行、物流、市场和区域数据 |
| 数据指标 | 大数字、趋势、条形或环形图 | KPI、市场、增长和统计摘要 |
| 决策 | 流程树、条件分支或比较矩阵 | 选择器、排查、判断路径 |

先选信息结构，再选择颜色、图标和装饰。

## 信息层级

默认结构：

```text
标题与结论
→ 核心结构或主数据
→ 分组模块
→ 解释和证据
→ 来源、注释或行动建议
```

每个模块只承担一个主要信息任务。

## 模块与网格

- 模块数量应受控；
- 同级模块保持统一尺寸、间距和标题位置；
- 主模块可以通过尺寸、颜色或位置放大；
- 阅读顺序通常从左到右、从上到下或沿箭头路径；
- 卡片不应平均化所有信息；
- 长图需要首屏结论和段落节奏。

## 图标与插图

- 图标必须代表具体概念，不做无意义装饰；
- 同一张信息图保持统一图标风格、线宽和透视；
- 复杂概念可使用小型场景插图，但不能抢过数据和结论；
- 人物、产品和设备示意要保持比例与身份一致；
- 箭头只用于表达方向、流程或因果，不要四处添加。

## 数据表达

- 数字本身应成为视觉元素；
- 图表类型要匹配数据关系；
- 比例、趋势和对比不能被装饰夸大；
- 不生成无法核实的具体数据；
- 用户未提供数据时，使用占位字段或示意值，并明确说明；
- 复杂精确图表适合后期排版，不依赖一次生图完成。

## 色彩系统

- 使用一套主色、辅助色和强调色；
- 同一类别使用一致颜色；
- 强调色只突出关键结论、异常或行动点；
- 背景保持足够平整，保证文字和图表可读；
- 不用过多渐变、发光和阴影制造“科技感”；
- 无障碍需求下注意红绿等颜色区分问题。

## 文字控制

- 每个模块只保留短标题和必要解释；
- 长段正文拆成要点；
- 数据、单位和时间格式统一；
- 标题、模块名、正文和注释形成稳定字号层级；
- 文字准确性要求较高时，优先生成无字结构图，再做后期文字排版。

## Prompt 结构

```text
生成一张 [信息图类型]，主题为 [主题]，
核心结论是 [一句话结论]，
采用 [流程 / 时间线 / 对比 / 层级 / 分类 / 数据] 结构，
阅读顺序为 [方向]，
主模块包含 [字段]，辅助模块包含 [字段]，
使用统一 [网格、图标和色彩系统]，
重点数据通过 [大数字 / 图表 / 强调色] 突出，
背景简洁，文字区平整，避免无意义装饰和虚构数据。
```

## 常见失败与修复

- 图标平均排列但没有逻辑：先确定流程、对比或层级关系；
- 模块太多：合并同类信息并突出结论；
- 箭头混乱：只保留表达真实方向和因果的箭头；
- 数据看似专业但无来源：改为用户提供数据或占位字段；
- 卡片风格不统一：统一尺寸、间距、标题和图标系统；
- 长图首屏无重点：把结论、标题和核心数据前置；
- 文字乱码：减少生图文字量，改用后期排版。
