# 等距微缩世界与剖面场景

## 解决什么问题

定义等距微缩城市、功能空间剖面、模拟经营场景、建筑模型和高信息密度世界观总览的实现方式，使空间、功能区和道具关系清楚，而不是只堆满可爱物件。

## 什么时候读取

- 用户要求 isometric、微缩世界、剖面建筑、模拟经营场景、基地总览或建筑模型；
- 场景需要同时展示多个楼层、功能区和连接结构；
- 需要从整体轮廓逐步阅读到局部道具；
- 高信息密度场景出现尺度混乱、区域无功能或背景抢主体。

## 核心公式

```text
稳定等距视角
+ 统一尺度
+ 清楚功能分区
+ 可用连接结构
+ 主题化道具
+ 主体突出、背景退让
+ 高密度但可读
```

## 视角与空间

- 使用约 30–45 度斜俯视或接近正交投影的三分之四视角；
- 垂直线尽量保持稳定；
- 不使用强广角和夸张透视；
- 建筑、人物、车辆和道具共享统一尺度；
- 微缩模型边缘、底板或剖面厚度应清楚；
- 场景中心可以是地标、核心设备或主功能区。

## 空间类型

### 等距城市模型

- 中心地标建立第一视觉点；
- 道路、水体、桥梁、绿地和建筑形成清楚网络；
- 小型车辆和人物提供尺度；
- 城市密度高，但主地标周围需保留呼吸空间。

### 建筑剖面与功能空间

- 显示楼层、房间、平台和内部结构；
- 每个区域有明确用途；
- 楼梯、电梯、走廊、桥梁、管线或传送带建立连接；
- 空间逻辑优先于装饰数量；
- 剖面切口和外壳边缘保持清楚。

### 模拟经营 / 游戏场景

- 区域像可点击、可操作和可探索；
- 道具与功能匹配；
- 入口、工作区、仓储、休息区和能源区等形成系统；
- 角色数量少而有行为，不做随机人群填充；
- 视觉阅读路径能沿道路、楼梯或工作流程移动。

### 地图与地形模型

- 2D 地图平面与 3D 地形或建筑体量对应；
- 等高线、道路、水系和边界清楚；
- 模型从底板“生长”出来，但不破坏地图关系；
- 注释和图例属于信息图或 graphic design，不在本页展开。

## 功能分区

每个区域至少回答：

- 用途是什么；
- 与哪个区域相连；
- 有哪些核心道具；
- 角色如何进入和使用；
- 是否属于公共、生产、展示、储存、生活或能源区域；
- 哪个结构决定辨识度。

高密度不等于每个角落都塞满物体。

## 道具与细节

- 道具必须与主题和功能相关；
- 同类道具保持统一尺寸和材质；
- 机器、桌椅、容器、植物和标识按区域分布；
- 关键设备可用强调色突出；
- 小物件需要可识别轮廓，避免变成噪点；
- 不为了“丰富”随机加入与世界观无关的物件。

## 风格方向

### 清洁卡通微缩

- 干净线条；
- 轻柔色块；
- 低到中饱和；
- 轻量阴影；
- 圆润边缘；
- 适合科普、模拟经营和儿童内容。

### 写实建筑模型

- 材质、道路、水体和建筑反射更真实；
- 使用柔和棚拍或建筑可视化灯光；
- 底板、模型边缘和微缩尺度明确；
- 避免真实城市照片感破坏模型感。

### 科幻或奇幻基地

- 世界观结构仍需功能合理；
- 能源、管线、交通和生活区域有因果关系；
- 发光和机械细节只用于说明功能；
- 不把科幻元素随机堆满所有区域。

## 色彩与灯光

- 主色统一整体世界；
- 辅助色区分功能区；
- 强调色用于关键设备、入口和路径；
- 背景低对比、低细节；
- 阴影保持轻量和方向一致；
- 写实模型可增加材质高光，但不压暗内部区域；
- 多层剖面要保证每层亮度可读。

## Prompt 结构

```text
生成一张 [等距城市 / 建筑剖面 / 模拟经营场景 / 地形模型]，
使用约 [30–45 度] 斜俯视和稳定正交感透视，
主体由 [功能区域] 组成，各区域通过 [楼梯 / 道路 / 平台 / 管线] 连接，
统一建筑、人物、车辆和道具尺度，
每个区域放置与功能相关的核心道具，
视觉阅读从 [整体轮廓] 进入 [功能区] 再到 [局部细节]，
背景降低对比，主体突出，保持高信息密度但不杂乱。
```

## 与相邻风格的边界

- 普通建筑渲染强调单一空间和真实视角；本页强调等距总览、多区域和系统关系；
- 信息图负责文字、标注和数据层级；本页负责空间模型本身；
- 产品爆炸图展示单一产品部件；本页展示场景和建筑系统；
- 纯平面等距图标不一定需要本页，除非存在完整微缩空间。

## 常见失败与修复

- 区域很多但无用途：为每区定义功能和连接关系；
- 道具随机堆叠：只保留服务主题和功能的物件；
- 等距透视漂移：固定相机角度并减少广角感；
- 人物和建筑尺度不一致：设置统一尺度参照；
- 剖面内部太暗：减弱真实阴影，保证每层可读；
- 背景抢主体：降低背景对比、饱和度和细节；
- 画面太满：减少重复道具，强化路径和主地标。
