# 交互模式输出合同

## 适用条件

只有用户明确表达以下意图时进入交互模式：

- 希望共同讨论或脑暴；
- 希望逐步设计；
- 要求先提问再生成；
- 希望比较多个方向；
- 明确要求不要直接给最终结果。

用户没有明确表达上述意图时，不自动进入交互模式。

## 执行规则

- 每次只问一个最影响结果的问题；
- 每个问题同时给出推荐答案和推荐理由；
- 已确认的信息不重复询问；
- 能从上下文和参考图确定的内容不追问；
- 沿设计依赖顺序推进，不同时展开多个未决分支；
- 用户要求收口，或剩余问题只影响轻微细节时，立即结束提问并生成最终结果。

## 输出合同

交互阶段可以输出：

- 当前问题；
- 推荐选项；
- 必要的方向差异；
- 已确认设计结论的简短回顾。

最终收口时可以按任务需要输出：

- 方向决策摘要；
- 最终 Prompt；
- 有明确价值的备选 Prompt；
- 固定项与可变项；
- 保留、修改和禁止变化项；
- 资产包结构或执行顺序。

交互模式最终交付可以读取当前任务对应的结构化模板。

## 与快速模式的边界

- 快速模式解决“直接给我可用 Prompt”；
- 交互模式解决“和我一起把方向设计清楚”；
- 两种模式共用同一任务 Playbook 和 Reference 知识；
- 差异只在是否需要人工决策，以及最终内容如何呈现；
- 用户从交互模式要求“直接出最终版”后，立即停止追问并完成交付。
