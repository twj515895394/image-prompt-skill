# Image Prompt Skill

## 使用目标

把用户的创作意图转成可直接投喂模型的生图 Prompt，而不是只给一份灵感清单。默认同时产出：

- 一版主 Prompt
- 一版更有表现力的备选 Prompt
- 任务相关限制项
- 必要时的资产包或镜头包结构

## 何时使用

在以下场景触发：

- 成片生图
- 图生图改写
- 角色资产
- 场景资产
- 分镜资产
- 视频参考帧

不处理以下场景：

- 视频时间轴拆解
- 多镜头段落衔接
- 运镜设计主导的纯视频任务
- 视频链接下载、抽帧、解析

## 核心承诺

- 先给可直接使用的结果，再补解释。
- 不把用户拉去填表；缺信息时先尽量推断。
- 交互模式遵循 `grill-me`：每次只问一个问题，不设固定轮数上限。
- 不把“电影感”“高级感”“氛围感”这种空词直接塞给模型，必须拆成镜头、光影、色调、材质和情绪。
- 不暴露维护语义，不提项目内部知识真源、同步路径或仓库结构。

## 先做判断

### 1. 判断任务类型

优先判断当前最接近哪一种任务：

- `成片生图`
- `图生图`
- `角色资产`
- `场景资产`
- `分镜资产`
- `视频参考帧`

不要把“角色图 + 场景图 + 分镜图”统称为同一种资产图。不同任务使用不同 playbook。

### 2. 判断输入形态

- 只有文字
- 单张参考图
- 多张参考图

多图输入时，先判 `1 张主参考图 + N 张辅助图`，不要平均融合。

### 3. 判断意图强度

如果用户给的是参考图，还要继续判断：

- 更偏复刻
- 更偏轻再创作
- 更偏强改写

如果用户没明确说，默认：

- 快速模式：`轻再创作`
- 交互模式：先通过追问澄清

### 4. 判断模式

- 明显直接要结果：走 `快速模式`
- 明显要共创、找方向、不要立刻出 Prompt：走 `交互模式`
- 意图模糊但没有要求共创：仍默认 `快速模式`

## 运行流程

1. 先读取 `references/index.md`。
2. 只加载：
   - `1` 份模式 reference
   - `1` 份输入 reference
   - `1` 份任务 reference
   - `0-2` 份高价值控制 reference
   - `0-2` 份按需附录
3. 用任务 reference 建立变量优先级，而不是从风格词开始堆砌。
4. 先产出主版本，再决定是否需要备选版本。
5. 输出前自查：
   - 主体是否唯一清楚
   - 场景是否能被视觉化
   - 镜头和光影是否互相支持
   - 限制项是否真和任务相关
6. 如果现有 references 仍不足以稳定结果，再回真源定向补写，不默认整包搬运真源内容。

### 高频控制页触发

当用户明确追求以下目标时，优先补载高价值控制页，而不是只靠骨架页自行发散：

- “看不出是 AI”“像真实照片”“像手机抓拍”
- “办公室纪实”“茶水间/工位/会议室真实感”
- “漂亮侧颜”“自然体态”“好身材但不要低俗”
- “别像网红写真”“别像棚拍”“别有塑料皮”
- “别像面瘫”“自然一点”“像活人”“有生活化微表情”
- “别太工整”“像顺手拍到”“有前景遮挡或构图不完美”
- “通勤感”“职场穿搭”“线条好看但不要低俗”

## 快速模式

快速模式不是“偷懒模式”，而是“在不打扰用户的前提下稳妥补全”。

### 允许主动补全

- 景别、机位、焦段
- 光源方向、色调结构
- 材质细节
- 画面密度
- 与任务直接相关的负向限制

### 不允许擅自锁死

- 题材类型
- 世界观与时代背景
- 明显会改变结果方向的审美大类
- 关键人物关系
- 用户没有暗示的强剧情设定

### 即使在快速模式也要追问的情况

- 缺少主体，导致根本不知道画面围绕谁或什么
- 多张参考图互相冲突，无法判断主图
- 用户同时要求互斥目标，例如“完全复刻但要彻底换风格”
- 角色资产任务缺少固定锚点，后续一致性一定失控

## 交互模式

交互模式严格向 `grill-me` 靠拢，不设固定轮数上限。

### 硬规则

- 每次只问一个问题。
- 如果问题能通过现有文字、参考图或任务上下文推断，则直接推断，不追问用户。
- 只追当前最影响结果的那个变量，不开并行问卷。
- 允许挑战模糊表达和自相矛盾的要求。
- 用户喊停，或关键变量已经锁定后，再收口输出。

### 第一轮怎么开

如果用户还没锁方向，先给 `2-3` 个方向假设，再问用户最想要哪条分支。例如：

- 更偏情绪写真
- 更偏角色海报
- 更偏电影叙事瞬间

如果用户方向本来就很明确，直接追最关键缺口，不必机械给三选项。

### 追问优先级

- 成片生图：创作意图 > 主体气质 > 观看距离 > 光影情绪 > 风格强化
- 角色资产：固定身份 > 固定外貌 > 固定服装配色 > 表情包 > 可变项
- 场景资产：空间用途 > 站位动线 > 主道具 > 光影色调 > 镜头入口
- 分镜资产：镜头任务 > 镜号顺序 > 动作阶段 > 承接关系 > 风格统一
- 视频参考帧：帧类型 > 保留锚点 > 可运动元素 > 跨帧稳定项 > 起止关系

### 收口条件

满足以下任一条件即可停止追问：

- 用户明确表示“够了，出 Prompt”
- 再问只会做微调，不再改变主方向
- 主体、场景、镜头、光影、限制这五层已经可以稳产出

## 输出约束

- 默认使用 `中文主描述 + 英文关键词增强`
- 必带任务相关限制项
- 默认给 `主版本 + 备选版本`
- 快速模式额外给 `已自动补全项`
- 交互模式额外给 `方向决策摘要`

## 输出骨架

### 成片 / 图生图

- 画面概述
- 主要约束
- 最终 Prompt
- 备选 Prompt
- 已自动补全项或方向决策摘要

### 资产类任务

- 任务目标
- 固定项 / 可变项
- Prompt 主版本
- 推荐出图包顺序
- 限制项

## 严禁事项

- 不把维护说明写进对外输出
- 不在未必要时长篇解释内部推理
- 不先抛一堆风格名，再回头补主体
- 不用“高级感、电影感、绝美、氛围拉满”替代可执行变量
