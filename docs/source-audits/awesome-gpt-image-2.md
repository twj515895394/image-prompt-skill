## 2. 许可与署名要求

仓库许可证允许分享和改编，但要求：

- 提供适当署名；
- 提供许可证链接；
- 说明是否进行了修改；
- 不暗示原作者或许可方为本项目背书。

投稿规范还要求记录原作者和原始来源。

本项目执行规则：

- 直接引用或高度接近原文时保留作者和原始来源；
- 规则提炼注明已改写、结构化或抽象；
- 不批量复制来源图片；
- 单个案例缺少作者或原始来源时，不沉淀完整 Prompt；
- 不删除署名后重新包装为本项目原创案例；
- 继续注意图片、品牌、肖像和商标等附加权利。

## 4. 适合沉淀的知识类型

- 产品展示类型、Hero 构图、棚拍灯光和材质表现；
- 海报、文字、缩略图和信息图的视觉层级；
- 漫画分格、故事板版式与跨格连续性；
- 摄影和电影画面的镜头、光影、色彩和材质规律；
- 插画、3D 和传统媒介的稳定实现方法；
- 可替换槽位清楚的 Prompt 结构；
- 多案例重复出现的失败风险和修复方式。

## 5. 不适合直接沉淀的内容

- 全量 Prompt 集合；
- 单条案例的完整创意和文案；
- 外部网站分类的原样复制；
- GPT Image 2 营销描述或未经验证的能力断言；
- 模型专属参数和偶然语法；
- 仅由一个案例支持的风格结论；
- 来源不完整的完整 Prompt；
- 大量生成图片本体。

## 6. 抽样策略

每个批次执行：

```text
初筛 20–30 个案例
→ 深读 8–12 个代表案例
→ 提炼 3–7 个稳定知识单元
→ 正式产出通常 1–5 个叶子或现有叶子增补
```

抽样应覆盖不同主体、构图、光影、材质、文字密度、写实度和失败风险。
