## 3. 来源与权利边界

这些文件由仓库维护者直接提供，可作为本项目的摄取输入。

但部分正文保留了第三方文章、图片或历史整理资料的“来源证据”，因此：

- 不默认这些第三方内容都可原样重新发布；
- 来源不完整时，只提炼通用方法和结构；
- 不直接搬运完整案例 Prompt、长段原文或图片；
- 对高度接近原文的内容，正式迁移前继续核对作者、来源和许可；
- 正式 Reference 应优先使用重新组织后的中文规则表达。

## 4. 适合沉淀的知识类型

- 可跨任务复用的镜头、光影、色调和 Prompt 组装规则；
- 妆容、发型、表情等高价值选择器的缺失项；
- 角色、场景和分镜 Playbook 的执行字段与稳定流程；
- 图像风格提取和参考图反推的方法；
- 写真、电影画面和生活化纪实风格的实现规律；
- 高频失败模式与修复方式。

## 5. 不适合直接沉淀的内容

- 与现有正式叶子重复的完整正文；
- 同时混合 task、control、library 和 style 职责的万能页；
- 只适用于单个案例的完整 Prompt；
- 缺少清晰路由条件的风格长名单；
- 无法核实来源的第三方原文或图片；
- 已被现有 Playbook 或选择器充分覆盖的重复知识。
