# 基础 Prompt 结构

## 定位

这是生图与生视频都可复用的最小结构。它连接 [[人物外貌与写真Prompt结构]]、[[图生视频Prompt结构]] 和 [[文生视频Prompt结构]]。

## 8 要素

| 要素 | 用途 | 写法要点 |
|---|---|---|
| 主体 | 确定画面中心 | 人物、物体、群像或场景核心，只保留一个主目标 |
| 场景 | 给模型空间上下文 | 地点、时代、天气、室内外、道具关系 |
| 动作 | 让画面发生变化 | 姿态、行为、互动、表情变化 |
| 镜头/景别 | 控制观看距离 | 远景、中景、近景、特写、焦段、机位 |
| 光线 | 决定质感 | 自然光、侧光、逆光、伦勃朗光、明暗对比 |
| 风格 | 统一审美方向 | 写实、电影感、胶片、动画、国漫、广告摄影 |
| 比例 | 适配平台与构图 | 1:1、3:4、9:16、16:9 等 |
| 限制 | 防止跑偏 | 不要多手、不要变脸、不要文字、水印、畸形等 |

## 写作顺序

1. 先写“主体 + 场景”，避免模型不知道重点。
2. 再写“动作 + 镜头”，让画面有事件与观看方式。
3. 再写“光线 + 风格”，统一质感。
4. 最后写“比例 + 限制”，用于控制输出边界。

## 可复用模板

```text
主体：[人物/物体/场景核心]
场景：[地点、时间、天气、空间关系]
动作：[正在做什么、表情或互动]
镜头：[景别、机位、焦段、构图]
光线：[主光、辅光、阴影、氛围]
风格：[媒介风格、质感、审美方向]
比例：[画幅比例、平台适配]
限制：[避免的错误、不要出现的元素]
```

## 检查表

- 主体是否唯一清晰。
- 场景是否能被视觉化，而不是只写抽象情绪。
- 镜头是否包含景别或机位，避免只写“高级感”。
- 风格是否服务主题，避免多个互相冲突的风格堆叠。
- 限制是否针对真实失败模式，而不是堆满无关负面词。

## 常见失败模式

- 只写风格名，不写主体和事件。
- 同时要求多个冲突镜头，例如“超远景特写”。
- 光线、色调、风格互相打架。
- 限制词过多，反而稀释正向目标。
