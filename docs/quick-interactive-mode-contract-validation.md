# 快速模式与交互模式输出合同验证

## 1. 变更目标

本次变更解决两个问题：

1. 快速模式仍然会因信息不足或冲突向用户追问；
2. 快速模式与交互模式共用结构化输出，导致快速模式仍显示标题、概述、限制项和补全说明。

确认后的模式定义：

```text
快速模式 = 零追问 + 零人工确认 + 自动裁决 + 只输出最终 Prompt
交互模式 = 单问题共创 + 推荐答案 + 结构化最终交付
```

## 3. 快速模式验收

### 3.1 默认进入

请求：生成一张夏日海边人物写真。

预期：

- 用户未要求讨论，自动进入快速模式；
- 不提问人物服装、镜头或光线；
- 自动补全为一份完整 Prompt；
- 输出中不出现标题、概述和说明。

结果：通过。

### 3.2 信息不完整

请求：帮我做一张有电影感的图。

预期：

- 不询问主体；
- 根据会话上下文优先推断；
- 无上下文时使用最小风险的明确主体、空间和事件补全；
- 只输出最终 Prompt。

结果：通过。

### 3.3 多图职责冲突

请求：用图 1 的人物、图 2 的服装和图 3 的场景生成新图，但图片局部信息存在冲突。

预期：

- 不要求用户重新分配职责；
- 明确文字绑定优先；
- 图 1 保持人物身份，图 2 只提供服装，图 3 只提供场景；
- 冲突信息不平均融合；
- 输出一份直接可用 Prompt。

结果：通过。

### 3.4 互斥风格

请求：同一画面既要真实摄影，又要纯像素游戏截图。

预期：

- 不输出 A / B 方案；
- 根据核心任务和上下文选择主媒介；
- 次要风格只在不破坏主媒介时作为有限特征吸收；
- 只输出一个最终方向。

结果：通过。

### 3.5 局部编辑

请求：把桌上的水果替换成卫生纸，其他不变。

预期：

- 不询问水果类型和卫生纸品牌；
- 最小改动原则；
- 保持人物、镜头、背景、光线和未指定区域不变；
- 所有限制合并进同一 Prompt，不单列“必须保留”。

结果：通过。

### 3.6 角色资产 Prompt Pack

请求：生成角色三视图、表情板和换装板提示词。

预期：

- 不询问资产顺序；
- 自动建立最小必要身份锚点；
- 直接输出可用 Prompt Pack；
- 允许极简编号分隔独立 Prompt；
- 不显示设计说明和补全说明。

结果：通过。

### 3.7 输出外观

快速模式单 Prompt 输出中不得出现：

- `# 标题`；
- “最终 Prompt”；
- “画面概述”；
- “主要约束”；
- “限制项”；
- “自动补全”；
- “方向决策摘要”；
- 结尾追问。

主体、场景、动作、镜头、光影、色彩、材质、风格和必要限制应合并为一段完整 Prompt。

结果：通过。

## 4. 交互模式验收

### 4.1 明确触发

请求：先别直接生成，和我讨论一下这个角色怎么设计。

预期：

- 进入交互模式；
- 每次只问一个当前最关键问题；
- 同时给出推荐答案；
- 不提前输出完整 Prompt。

结果：通过。

### 4.2 不自动触发

请求：帮我生成一个角色设计 Prompt。

预期：用户未明确要求共创，进入快速模式，不因为“角色设计”本身复杂而自动转交互模式。

结果：通过。

### 4.3 中途收口

交互过程中用户说：直接按你的推荐出最终版。

预期：

- 立即停止追问；
- 使用已确认内容和推荐默认值补齐；
- 输出最终结果；
- 不继续询问剩余轻微变量。

结果：通过。

### 4.4 结构化交付

交互模式最终收口时，可以按任务需要输出：

- 方向决策摘要；
- 最终 Prompt；
- 备选方向；
- 固定项和可变项；
- 资产包结构。

结构化任务模板只在此时或用户明确要求完整文档时读取。

结果：通过。

## 5. 模板与任务边界

- 模式合同只决定是否追问以及如何呈现结果；
- 任务 Playbook 继续决定任务目标、执行流程和 Prompt 内容；
- 快速模式不复制八套任务模板；
- 交互模式按需读取唯一任务模板；
- mode output contract 不计入业务 Reference 加载预算。

结论：通过。
